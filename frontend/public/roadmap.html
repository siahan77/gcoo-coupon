📋 상품 구성품(bundle_item) 지급 정책
  • 지급 수량 결정 방식
    FIXED (고정): 정해진 수량으로 지급
    AUTO (자동계산): 기간 또는 조건에 따라 자동 계산
  • 구독 유상(base) / 무상(bonus) 분리
    기준: base_cycle_days=28
    유상(base) 부분은 요금에 비례하여 지급
    무상(bonus) 부분은 추가 혜택으로 지급
  • 이용 방식
    단일 소진권(구 1회권): 1회 사용으로 전체 소진
    일일 갱신권(구 정기권): 매일 새로운 량이 갱신

    미사용 일수에 대해 unit_price만큼 환불

📡 쿠폰 발급 채널
  • 상품 구매 (PRODUCT_PURCHASE)
    상품 구매 시 자동으로 쿠폰 발급
  • 시스템 트리거 (SYSTEM_TRIGGER)
    특정 조건 만족 시 자동 발급 (이벤트, 마일리지 등)
  • 쿠폰 코드 (COUPON_CODE)
    사용자가 코드 입력으로 쿠폰 수령
  • 관리자 직접 지급 (ADMIN_GRANT)
    관리 시스템에서 수동으로 사용자에게 지급
📄 쿠폰 정책
🔢 쿠폰 계산 단계 (PRE/POST)
  • PRE 단계 할인
    적용 시점: 요금 산정 직후
    대상: 시간패스, 거리패스, 정액할인, 결제상한
  • POST 단계 할인
    적용 시점: 전처리(PRE) 할인 적용 이후, 최종 결제 직전
    대상: 정률할인
📄 쿠폰 계산 단계 정책
📐 쿠폰 할인 구조 (4-axis)
  • Scope (적용 범위)
    UNLOCK (잠금해제), RIDE (탑승료), ORDER (주문 전체)
  • Basis (할인 기준)
    NONE (조건없음), TIME (시간 기반), DISTANCE (거리 기반)
  • Unit (할인 단위)
    PERCENT (정률/%), AMOUNT (정액/금액)
  • Apply Rule (적용 규칙)
    eligibility (자격조건), window (사용 구간), cap (상한선)
📄 쿠폰 할인 기준 산식
🌐 쿠폰 사용 국가 정책
  • 국가 일치 조건
    서비스국가 + 회원국가 + 쿠폰상태 모두 일치 필요
  • 국가 불일치 시 처리
    비활성 (INACTIVE): 쿠폰 사용 불가
    비노출 (HIDDEN): 사용자 UI에서 보이지 않음
📄 쿠폰 정책
📊 쿠폰 노출 순서
  • 5단계 우선순위
    1순위: 사용중 쿠폰
    2순위: 다회성 시간패스
    3순위: 적립성 시간패스
    4순위: % 쿠폰 (정률 할인)
    5순위: 금액 할인 쿠폰
